Retail Management System

About Project

This was a group project I worked on in my second year of university. I was appointed as the SCRUM Master, which meant I was responsible for helping the team stay organised and on track with deadlines, facilitating our Agile process, running stand-up meetings, and making sure we kept clear communication throughout the project.

The goal of the project was to build a system that supports several different user types:

  • Customers can view product availability in a specific branch.
  • Cashiers can process in-person orders, which automatically generate and email invoices to customers while updating stock levels.
  • Inventory Managers can add new products and adjust stock levels at different locations.
  • Admins have access to full reporting tools, including stats like most popular products and total earnings, and can manage staff accounts.

We designed and built the system using the MVC architecture, with Laravel for the backend and Bootstrap for the frontend to keep things responsive and user-friendly.

Key Features

  • Product stock lookup by branch for customers
  • Order processing and automated invoicing for cashiers
  • Role-based access for inventory managers and admin
  • Live stock updates and inventory management tools
  • Admin dashboard with analytics and user management
  • Built using Laravel (PHP), Bootstrap, and MySQL
